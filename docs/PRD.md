# GOD SLAYER — PRD v0.1

> Skul: The Hero Slayer에서 영감을 받은 **로그라이트 벨트스크롤 액션**  
> 플레이어는 신(神)들의 두개골을 빼앗아 장착하고, 그 힘으로 신들을 처치한다.

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 로그라이트 · 벨트스크롤 액션 |
| 엔진 | Godot 4.7 (2D) |
| 시점 | 횡스크롤 고정 카메라 |
| 플랫폼 | PC (Windows) |
| 목표 플레이타임 | 런 1회 40~60분 |

### 한 줄 컨셉
> *"신의 두개골을 훔쳐 신이 되어라"*  
> 죽은 신의 두개골을 장착해 능력을 획득하고, 스테이지를 돌파하며 최종 신을 처치한다.

## 2. 핵심 메카닉

### 2-1. 두개골 시스템 (Skull Swap)
Skul의 핵심 아이덴티티. 이 게임에서 두개골 = 신의 잔재.

- 플레이어는 **두개골 2개**를 동시에 소지
- `Q` / `E` 또는 자동으로 **스왑** 가능
- 각 두개골마다 **기본 공격(J)** + **스킬(K)** + **패시브** 보유
- 두개골에는 **레어도** 존재: 일반 → 희귀 → 전설

| 두개골 예시 | 기본 공격 | 스킬 | 패시브 |
|------------|----------|------|--------|
| 전사신의 두개골 | 3연속 검격 | 돌진 베기 | 처치 시 HP 회복 |
| 마법신의 두개골 | 마력 구체 | 광역 폭발 | 마법 쿨다운 -20% |
| 속도신의 두개골 | 빠른 연타 | 잔상 대시 | 이동속도 +30% |
| 죽음신의 두개골 | 낫 휘두르기 | 영혼 소환 | HP 30% 이하 시 공격력 2배 |

### 2-2. 이동 조작
벨트스크롤 기반, 수직 요소 포함.

| 키 | 동작 |
|----|------|
| A / D | 좌우 이동 |
| S | 숙이기 (회피, 낮은 공격 판정) |
| W | 사다리 타기 / 위 플랫폼 올라가기 |
| J | 기본 공격 (장착 두개골 기준) |
| K | 스킬 (쿨다운 존재) |
| Space | 점프 / 이단 점프 |
| Shift | 대시 (무적 프레임 포함) |
| Q | 두개골 스왑 |

### 2-3. 로그라이트 루프

```
[시작] 허브 → [스테이지 진입] → 방 선택 → 전투/이벤트/상점
       ↑                                          ↓
    [사망 시]                              [보스 처치 → 다음 스테이지]
  EXP 일부 유지                              ↓
  허브 영구 강화                        [최종 보스 처치 → 클리어]
```

### 2-4. 방 선택 시스템 (Skul 스타일)
각 스테이지는 **노드 맵** 구조. 분기 선택으로 루트가 달라진다.

```
        [전투]─[상점]
       /              \
[시작]                 [보스]
       \              /
        [이벤트]─[전투]
```

방 타입:
| 타입 | 내용 |
|------|------|
| ⚔ 전투 | 적 처치 후 두개골/아이템 드랍 |
| 💀 엘리트 | 강한 적 1마리, 희귀 드랍 보장 |
| 🏪 상점 | 골드로 두개골/유물 구매 |
| ✨ 신의 제단 | 두개골 강화 또는 교체 |
| 📖 이벤트 | 선택지형 이벤트, 리스크·리워드 |
| 💛 보스 | 스테이지 보스, 전설 두개골 드랍 |

## 3. 진행 구조

### 스테이지 구성 (총 4스테이지)

| 스테이지 | 배경 | 보스 | 특징 |
|---------|------|------|------|
| 1 | 폐허의 신전 | 수호신 | 튜토리얼, 기본 적 |
| 2 | 타락한 숲 | 자연신 | 원거리 적, 함정 |
| 3 | 천공의 성채 | 전쟁신 | 고속 적, 다층 구조 |
| 4 | 신들의 좌 | 최고신 | 2페이즈 보스 |

### 영구 강화 (허브)
런 종료 후 획득한 **신의 기억(영혼 파편)** 으로 강화.

| 강화 항목 | 효과 |
|----------|------|
| 최대 HP | +10%/단계 |
| 기본 공격력 | +5%/단계 |
| 대시 횟수 | +1 (최대 2) |
| 골드 획득량 | +10%/단계 |
| 시작 두개골 선택지 | +1 (기본 3→최대 5) |

## 4. 유물 시스템 (Relic)
Skul의 아이템과 유사. 런 중 획득, 종료 시 초기화.

- 런 당 최대 **8개** 소지
- 두개골과 시너지 발생 시 **연계 효과** 활성화
- 예시:
  - *붉은 피의 반지* — HP 50% 이하 시 공격속도 +40%
  - *죽은 신의 눈* — 처치 시 5% 확률로 두개골 드랍
  - *시간의 모래시계* — 스킬 쿨다운 전체 -15%

---

## 5. 적 & 보스 설계 원칙

### 일반 적
- 각 스테이지마다 **6~8종** 적 배치
- **원거리형 / 근거리형 / 탱커형** 조합으로 방 구성
- 처치 시 골드·경험치·두개골 조각 드랍

### 보스
- **2단계 체력바** (50% 이하 시 패턴 변화)
- 처치 시 **전설 두개골 1개** 드랍 보장
- 보스는 처치한 신의 두개골이므로 스토리와 연결

## 6. 기술 스펙 (Godot 4.7)

### 씬 구조
```
Main.tscn
├── Stage (현재 스테이지 씬 인스턴스)
│   ├── Background
│   ├── Platforms (TileMapLayer)
│   ├── Enemies (Node2D)
│   └── Interactables
├── Player.tscn
│   ├── CharacterBody2D
│   ├── SkullHolder (두개골 2개 관리)
│   ├── HitboxComponent
│   └── Camera2D (수평 추적)
└── UI
    ├── HUD (HP바, 두개골 슬롯, 스킬 쿨다운)
    └── RoomMap (노드 맵 오버레이)
```

### 주요 시스템 클래스
| 클래스 | 역할 |
|--------|------|
| `SkullHolder` | 두개골 2개 슬롯 관리, 스왑 처리 |
| `SkullData` (Resource) | 두개골 스탯·스킬 정의 |
| `RunManager` (Autoload) | 현재 런 상태 (두개골, 유물, 골드) |
| `StageGenerator` | 방 노드 맵 생성 |
| `EnemyBase` | 적 AI 베이스 클래스 |
| `BossBase` | 보스 패턴 베이스 클래스 |
| `RelicManager` | 유물 효과 적용 |

### 데이터 관리
- 두개골 / 유물 / 적 스탯 → **Resource (.tres)** 파일로 관리
- 영구 강화 데이터 → `user://save.json`
